# 網站業主驗收測試流程 (Owner Acceptance Testing Guide)

本文件旨在協助業主進行 **Rentaloop** 網站的功能驗收測試。請依循以下步驟，確認各項核心功能運作正常。

此測試流程包含：
1.  **會員登入與註冊**
2.  **上架案件 (租賃品)**
3.  **租用與預約功能**
4.  **Admin 後台管理功能**

---

## 1. 會員登入與註冊 (Login & Registration)

### 測試目標
確認使用者可以透過 Google 帳號順利登入系統，並導向至會員中心。

### 測試步驟
1.  進入網站首頁，點擊右上角的 **「登入/註冊」** 按鈕，或直接進入登入頁面 (`/auth`)。
2.  您會看到「登入」與「註冊」的分頁，兩者皆支援 **Google 一鍵登入**。
3.  點擊 **「登入 Rentaloop.net」** (Google 按鈕)。
4.  選擇您的 Google 帳號進行驗證。
5.  **預期結果**：
    *   系統應成功驗證身分。
    *   畫面應自動跳轉至 **「會員中心」 (`/member`)**。
    *   右上角應顯示您的頭像或名稱。

---

## 2. 上傳案件 / 上架物品 (Upload Listing)

### 測試目標
確認「出租方 (Host)」可以順利填寫物品資訊並將物品上架至平台。

### 測試步驟
1.  確認您已登入，並位於 **「會員中心」** (`/member`)。
    *   *注意：若您的帳號尚未成為認證出租方，系統可能會提示您進行驗證 (視目前權限設定而定)。*
2.  點擊會員中心的 **「上架新物品」** 按鈕 (或前往 `/items/new`)。
3.  依序填寫以下欄位：
    *   **物品名稱**：輸入測試用的物品名稱 (例如：Canon R6 相機)。
    *   **類別**：從下拉選單選擇合適類別 (例如：攝影器材)。
    *   **物品狀況**：選擇狀況 (例如：全新、近全新)。
    *   **物品描述**：輸入簡單的描述文字。
    *   **物品照片**：點擊上傳區域，選擇一張或多張照片 (支援拖曳上傳)。
    *   **每日租金**：輸入金額 (例如：800)。
    *   **押金**：輸入金額 (例如：5000)。
    *   **最早可租借日期**：選擇日期。
    *   **面交地點**：輸入地點名稱 (例如：台北車站)。
    *   **使用說明/注意事項**：輸入測試文字。
4.  點擊下方的 **「發布物品」** 按鈕。
5.  **預期結果**：
    *   系統顯示成功訊息。
    *   回到會員中心後，應可在 **「已上架物品」** 列表中看到剛剛新增的物品。

## 3. 租用功能 / 預約流程 (Rental Process)

### 測試目標
確認「承租方 (Renter)」可以瀏覽物品、選擇日期並送出預約請求。

### 測試步驟
1.  **瀏覽物品**：
    *   前往 **「瀏覽全部」** 頁面 (`/products`)。
    *   找到您剛剛上架的測試物品，點擊進入 **物品詳情頁** (`/products/[您的物品ID]`)。
2.  **操作預約 (電腦版)**：
    *   在畫面右側的 **「租賃卡片」** 區塊，點擊日曆。
    *   選擇 **「起始日期」** 與 **「結束日期」**。
    *   確認下方顯示的「總金額」與「天數」計算正確。
    *   點擊 **「送出預約」** (若未登入會提示登入)。
3.  **確認預約**：
    *   系統會跳出確認視窗，顯示租金明細。
    *   點擊 **「確認預約」**。
4.  **預期結果**：
    *   顯示「預約成功」訊息。
    *   系統會自動通知出租方 (您可以在會員中心的 **「待處理預約」** 查看)。
    *   *註：若您是使用同一帳號測試，您同時是出租方也是承租方，系統可能會顯示「保留此時段」而非預約，建議使用無痕模式或另一帳號測試承租流程。*

## 4. Admin 端測試 (Admin Panel)

### 測試目標
確認網站管理員 (Admin) 可透過後台管理會員、訂單與網站內容。

### 測試步驟
1.  使用具有 Admin 權限的帳號登入。
2.  進入 **後台管理系統** (`/admin`)。
3.  依序點擊左側選單，測試各區塊是否能正常顯示資料：

| 功能選單 | 測試重點 |
| :--- | :--- |
| **總覽 (Dashboard)** | 確認是否顯示基本的統計數據（如有）。 |
| **預約訂單管理** | 查看所有租賃訂單的狀態，確認剛剛測試的訂單是否出現。 |
| **一般會員管理** | 查看所有註冊會員列表。 |
| **租貸會員審核** | 查看申請成為出租方的會員資料 (KYC 審核)。 |
| **上架物品管理** | 查看所有上架物品，測試是否能 **下架** 或 **刪除** 違規物品。 |
| **物品分類管理** | 測試是否能 **新增** 或 **編輯** 現有的物品分類標籤。 |
| **橫幅圖片管理** | 管理首頁輪播圖 (Banner)，測試上傳新圖片。 |
| **信件模板 (Resend)** | 查看系統發信紀錄或模板設定。 |
| **SEO 文章 (SDGs)** | 測試 **發布新文章** 功能，確認前台部落格是否更新。 |
| **首頁組件設定** | 調整首頁顯示的區塊與順序 (如有此功能)。 |

---

### 備註
*   如果在測試過程中遇到任何錯誤畫面 (Error Page) 或按鈕無反應，請截圖並記錄發生步驟，回報給開發團隊。
*   手機版測試建議使用手機實際操作，體驗動線是否順暢。
